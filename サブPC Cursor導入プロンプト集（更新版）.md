# サブPC Cursor導入プロンプト集（更新版）

## 🎯 概要

サブアカウントのサブPCのCursorで、マルチデバイス・マルチアカウントシステムを導入するための適切なプロンプト集です。

**⚠️ 重要更新**: 包括的なIssue管理システムが構築され、11個の完了Issueと体系的なラベル管理が実装されています。

## 📝 基本プロンプト（更新版）

### **完全導入プロンプト（更新版）**
```
以下のGitHubリポジトリをクローンし、マルチデバイス・マルチアカウントでのAI駆動開発統合システムを導入してください。

リポジトリURL: https://github.com/0nyx-lab/aidd-integration-system

## 導入要件

### 1. リポジトリのクローンとセットアップ
- リポジトリをクローン
- Python仮想環境の作成・有効化（Mac環境）
- 依存関係のインストール

### 2. Cursor設定の同期
- .cursor/rules/ ディレクトリの作成
- templates/cursor-rules/ からルールファイルをコピー
- デバイス固有の設定（Mac環境）の適用

### 3. マルチデバイス同期システムの設定
- multi_device_config.json の設定
- デバイス情報の登録（sub-pc, macOS, sub-account）
- 同期システムのテスト実行

### 4. 知識蓄積システムの統合
- knowledge.mdc でのデバイス固有知見の記録
- GitHub同期の設定
- 自動同期スクリプトの作成

### 5. Issue管理システムの活用（新規追加）
- 既存の11個の完了Issueの確認
- Issue #5（サブPC実装・テスト）の進捗報告
- 新たな課題発生時のIssue作成・管理
- ラベル体系の理解・活用

## 期待される結果
- メインPC（Windows）との知識共有が可能
- デバイス固有の知見を蓄積・共有
- 統合PDCAサイクルの実行
- 継続的な知識蓄積・学習システムの運用
- Issue管理システムによる体系的な課題管理

詳細な実装手順は、リポジトリ内の「マルチデバイス実装ガイド.md」を参照してください。
```
