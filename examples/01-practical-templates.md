# 実践テンプレート集 - AI駆動開発PDCAサイクル

## 概要

実際の運用で使用できるテンプレート集です。Issue #1の実装から得られた知見を基に作成されています。

## 1. Issue作成テンプレート

### 基本テンプレート
```markdown
## 背景 / 目的
[機能の背景と目的を記述]

## 要件
- [ ] [要件1]
- [ ] [要件2]
- [ ] [要件3]

## テスト観点
1. 正常系: [期待される動作]
2. 異常系: [エラーハンドリング]
3. 境界値: [境界条件のテスト]

## 完了条件（DoD）
- [ ] [完了条件1]
- [ ] [完了条件2]
- [ ] [完了条件3]
```

### 実際の使用例（Issue #1）
```markdown
## 背景 / 目的
練習として、最小のスクリプトを作成する。

## 要件
- [ ] 実行すると「Hello AIDD」と表示する

## テスト観点
1. 正常系: 実行結果が「Hello AIDD」である

## 完了条件（DoD）
- ターミナルで実行して「Hello AIDD」と表示される
- README に実行手順を追記
```

## 4. 環境設定テンプレート

### Windows環境
```powershell

py -m venv .venv

# 仮想環境の有効化
.\.venv\Scripts\Activate.ps1

# パッケージのアップグレード
python -m pip install -U pip

# スクリプトの実行
python src/[ファイル名].py

source .venv/bin/activate

# パッケージのアップグレード
python -m pip install -U pip

```

## 7. 実際の運用フロー

### 日常的な作業フロー
1. **作業開始**
   - 関連Issueの確認
   - ルールの確認宣言
   - 仮想環境の設定

2. **実装作業**
   - コードの作成
   - 基本動作の確認
   - 進捗の定期更新

3. **テスト・評価**
   - テストの実行
   - 結果の記録
   - 問題の特定

4. **完了処理**
   - 最終評価の実施
   - Issueの更新
   - 学習内容の反映

### 週次メンテナンス
1. **完了したIssueの分析**
2. **knowledge.mdcの更新**
3. **ルールの見直し**
4. **効果測定**

## 8. トラブルシューティング

### よくある問題と解決策

#### 仮想環境の問題
```markdown
## 問題: 仮想環境が認識されない
### 症状
- pythonコマンドがグローバル環境を参照
- パッケージがインストールされない

### 解決策
1. 仮想環境の再作成
2. パスの確認
3. 環境変数の設定確認

### 再発防止
- environment.mdcに仮想環境使用を明記
- 作業開始時の環境確認を必須化
```

#### GitHub MCPの問題
```markdown
## 問題: GitHub MCPが動作しない
### 症状
- Issueの作成・更新ができない
- 認証エラーが発生

### 解決策
1. Personal Access Tokenの確認
2. MCP設定の確認
3. 権限の確認

### 再発防止
- 定期的なToken更新
- 権限の最小化
```
