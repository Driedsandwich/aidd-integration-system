# Notion統合PIMS（無効化）

## 無効化理由（2025-10-12）

**エンタープライズレベルの運用体制を撤収**

### 過剰だった要素

- ポストモーテム運用
- SLO評価・監査ログ
- 運用手順書（runbook）
- GitHub Actions自動化
- 双方向同期ワークフロー

### 判断根拠

1. **非エンジニアには運用困難**
   - ポストモーテムの記述は専門知識が必要
   - SLO評価は個人開発に過剰
   - 運用手順書の保守コストが高い

2. **GitHub Issueで十分**
   - タスク管理
   - 進捗記録
   - PDCAサイクル

3. **複雑性のデメリット**
   - 同期エラー時の対処が困難
   - 設定・保守の学習コスト
   - 「管理のための管理」に陥る

## 保持する機能

**Notion MCP経由のシンプル活用**

ユーザーの要望に基づき、以下の機能は保持：

### データ貯蔵庫として活用
- GitHubリポジトリ内容の可視化
- プロジェクトメモの保存
- 知見の蓄積

### MCP経由の読み書き
- Cursor/ClaudeCodeから自然言語でアクセス
- 「このリポジトリは何？」→ Notionを確認
- 記録・検索・連携コンボ

### 設定方法

詳細は`docs/notion-mcp-simple-guide.md`を参照

## 将来の再評価条件

以下の状況が発生した場合は再検討：

1. **チーム開発への移行**
   - 複数人での情報共有が必要
   - 統一されたダッシュボードが必要

2. **エンタープライズレベルの需要**
   - SLO評価が必要な規模
   - ポストモーテムが必要な体制

3. **自動化の必要性**
   - GitHub Actions連携が必須
   - 双方向同期の明確な価値

## 保持理由

完全削除ではなく`.disabled/`に保持：
- 将来の参照用
- 設定情報の保存
- チーム開発への移行時の資産
