# マルチデバイス・マルチアカウントシステム導入案

## 🎯 概要

本システムは、複数のデバイス（メインPC、サブPC）と複数のアカウント（メインアカウント、サブアカウント）で、AI駆動開発の統合システムを運用するための導入案です。

## 🏗️ システム構成

### 1. デバイス構成

#### メインPC（Windows）
- **アカウント**: 0nyx-lab（メイン）
- **リポジトリ**: aidd-integration-system（オーナー）
- **権限**: Admin（完全管理）
- **役割**: システム管理・統合運用

#### サブPC（Mac）
- **アカウント**: サブアカウント
- **リポジトリ**: aidd-integration-system（コラボレーター）
- **権限**: Write（書き込み）
- **役割**: 並行開発・知識蓄積

### 2. リポジトリ構成

```
aidd-integration-system/
├── README.md                    # プロジェクト概要
├── docs/                        # ドキュメント（7ファイル）
├── src/                         # ソースコード（2ファイル）
├── templates/                   # テンプレート（5ファイル）
├── examples/                    # 実装例（2ファイル）
├── research/                    # 研究資料（2ファイル）
├── archive/                     # アーカイブ（12ファイル）
└── knowledge/                   # マルチデバイス知識蓄積
    ├── main-pc/                 # メインPCの知見
    ├── sub-pc/                  # サブPCの知見
    └── integrated/              # 統合知見
```

## 🔄 運用フロー

### 1. 日常運用

#### メインPC
1. **開発**: メイン機能の開発・統合
2. **知識蓄積**: knowledge.mdc での知見記録
3. **同期**: GitHubへの定期コミット・プッシュ
4. **統合**: サブPCからの知見の統合

#### サブPC
1. **開発**: 並行機能の開発・実験
2. **知識蓄積**: デバイス固有の知見記録
3. **同期**: GitHubへの定期コミット・プッシュ
4. **学習**: メインPCからの知見の学習

### 2. 知識統合フロー

```
メインPC知見 → GitHub → サブPC学習
サブPC知見 → GitHub → メインPC統合
統合知見 → GitHub → 全デバイス共有
```
