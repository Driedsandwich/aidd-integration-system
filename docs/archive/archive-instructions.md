# 既存ファイル格納手順書

## 📋 概要

既存の30程度のtxt/mdファイルを暫定的に格納し、その後構造化するための手順書です。

## 🎯 格納先

### **一時格納場所**: `docs/archive/raw-files/`
```
docs/archive/raw-files/
├── txt/          # .txtファイルの格納
├── md/           # .mdファイルの格納
└── other/        # その他のファイル形式
```

### **構造化後格納場所**: `docs/archive/categorized/`
```
docs/archive/categorized/
├── requirements/ # 要件定義関連
├── technical/    # 技術資料
├── templates/    # テンプレート
├── references/   # 参考資料
└── insights/     # 知見・ノウハウ
```

## 🔄 格納手順

### Step 1: 一時格納
```bash
# .txtファイルの格納
mv *.txt docs/archive/raw-files/txt/

# .mdファイルの格納
mv *.md docs/archive/raw-files/md/

# その他のファイルの格納
mv [その他のファイル] docs/archive/raw-files/other/
```

### Step 2: ファイル一覧作成
```bash
# 格納されたファイルの一覧を作成
find docs/archive/raw-files/ -type f > docs/archive/file-list.txt
```

### Step 3: 内容分析・分類
各ファイルの内容を確認し、以下の基準で分類：

#### 要件定義関連 (`requirements/`)
- プロジェクト要件
- 機能仕様
- 非機能要件
- 受け入れ条件

#### 技術資料 (`technical/`)
- アーキテクチャ設計
- 技術選定資料
- 実装ガイド
- パフォーマンス要件

#### テンプレート (`templates/`)
- 開発計画テンプレート
- ドキュメントテンプレート
- コードテンプレート

#### 参考資料 (`references/`)
- 技術文書
- ベストプラクティス
- 事例研究
- 外部リンク

#### 知見・ノウハウ (`insights/`)
- 開発経験
- 失敗例・解決策
- 効率化手法
- 学習メモ

### Step 4: 構造化・移動
```bash
# 分類に基づいてファイルを移動
mv docs/archive/raw-files/[ファイル] docs/archive/categorized/[カテゴリ]/
```

## 🚀 次のステップ

### Phase 1: 一時格納完了後
1. ファイル一覧の作成
2. 各ファイルの内容確認
3. 分類基準の適用

### Phase 2: 構造化完了後
1. 重複ファイルの統合検討
2. ファイル名の標準化
3. 内容の整理・統合

### Phase 3: 開発活用
1. ハッカソン開発プロセスでの活用
2. 必要に応じて `docs/requirements/` 等に移動
3. 継続的な更新・改善

## ⚠️ 注意事項

- **バックアップ**: 移動前に元ファイルのバックアップを取る
- **内容確認**: 移動前にファイル内容を確認
- **重複チェック**: 類似内容のファイルの統合を検討
- **機密情報**: 機密情報の取り扱いに注意
- **バージョン管理**: Gitでの変更履歴を適切に管理
