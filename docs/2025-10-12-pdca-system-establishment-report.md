##### **2.2 判断基準の定義**

**Issue化すべきタスク**:
```
✅ 30分以上の作業
✅ 複数ステップが必要
✅ 学習要素を含む
✅ 将来の参考になる可能性
```

**PDCA適用すべきケース**:
```
✅ 初回実施のタスク
✅ 失敗から学ぶべきタスク
✅ 標準化したいプロセス
✅ チーム共有が必要なタスク
```

##### **2.3 実行手順の標準化**

```
P (Plan): 計画・記録
├─ GitHub Issue作成
└─ Notion記録（必要に応じて）

D (Do): 実行
└─ 進捗をIssueにコメント

C (Check): 評価・学習
├─ 成功要因の特定
├─ 問題点の分析
└─ 改善点の抽出

A (Act): 改善・標準化
├─ ルール更新
├─ テンプレート作成
└─ 知識共有
```

## 🎓 重要な学習と洞察

### **1. メタレベルの思考**

```
レベル0: 実作業（Notion整理）
         ↓ PDCAサイクル適用
レベル1: プロセス確立（PDCAルール化）
         ↓ メタPDCAサイクル適用
レベル2: メタ学習（「ルール作り自体もIssue化すべき」）
         ↓ 検証と完全化
レベル3: 自己検証（報告内容の事実確認、Issue Close）
```

### **2. 自己進化システムの確立**

> **「PDCAサイクルを回すこと自体にもPDCAを適用する」**

これにより実現されること:
- ✅ プロセス自体が継続的に改善される
- ✅ 「どのようにルールを作るか」という知見が蓄積される
- ✅ すべての意思決定プロセスが透明化される
- ✅ 次回のルール策定時の参考になる
- ✅ **自己検証機能により、正確性が担保される**

### **3. 適用範囲の明確化**

**適用対象**:
- 技術的タスク（実装、設定、トラブルシューティング）
- 組織的タスク（整理、整理、ルール策定）
- 学習的タスク（新技術習得、ベストプラクティス導入）
- 運用タスク（ドキュメント更新、プロセス改善）

**判断基準による自動適用**:
- 30分以上 → Issue化
- 複数ステップ → PDCA適用
- 学習要素あり → 知見として記録
- 将来の参考 → テンプレート化
