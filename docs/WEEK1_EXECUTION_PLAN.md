## Day 1-2: Notion MCP設定（⭐⭐⭐⭐⭐ スコア4.7）

### 技術的根拠

**なぜ最優先か**:
1. **ユーザー要望に直接対応**: 「GitHubリポジトリが何なのか、ぱっと見て分かれば楽」
2. **即座の価値**: 設定後すぐに使える
3. **低コスト**: 20分の設定のみ
4. **低リスク**: 失敗しても影響なし
5. **動画思想に合致**: MCP活用そのもの

---

### 実行手順

#### Step 1: Notion Integration作成（5分）

```markdown
1. https://www.notion.so/my-integrations へアクセス
2. 「New integration」をクリック
3. 以下を設定:
   - Name: "Cursor AI Integration"
   - Type: Internal Integration
   - Associated workspace: 自分のワークスペース
4. 「Submit」をクリック
5. 「Internal Integration Secret」をコピー
   → これがAPI Key（NOTION_API_KEY）

保管: .env ファイルに追加
```

#### Step 2: Notionデータベース作成（5分）

```markdown
1. Notionで新規ページ作成
   - タイトル: "GitHub Projects Database"

2. データベースを作成（Table形式）

3. プロパティ設定:
   | Property | Type | 説明 |
   |----------|------|------|
   | Name | Title | プロジェクト名 |
   | GitHub URL | URL | リポジトリURL |
   | Description | Text | 説明 |
   | Type | Select | project/learning/experiment |
   | Status | Select | active/frozen/archived |

4. サンプルデータ入力（テスト用）:
   - Name: aidd-integration-system
   - GitHub URL: https://github.com/Driedsandwich/aidd-integration-system
   - Description: AI駆動開発の統合システム
   - Type: project
   - Status: active
```

#### Step 3: データベースをIntegrationに共有（1分）

```markdown
1. 作成したデータベースを開く
2. 右上「•••」→「Connections」
3. 「Cursor AI Integration」を選択
4. 「Confirm」
```

#### Step 4: Cursor MCP設定（5分）

```markdown
1. Cursorの設定を開く
   - Settings → Features → Beta

2. MCP設定ファイルを編集
   （ファイルの場所はOSによって異なる）
   
   Windows: %APPDATA%\Cursor\User\globalStorage\saoudrizwan.claude-dev\settings\mcp.json

3. 以下を追加:
```json
{
  "mcpServers": {
    "notion": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-notion"],
      "env": {
        "NOTION_API_KEY": "your-api-key-here"
      }
    }
  }
}
```

4. Cursorを再起動
```

#### Step 5: 動作確認（5分）

```markdown
1. Cursorで新しいチャットを開始

2. テストプロンプト:
「Notionに保存されているプロジェクト一覧を教えて」

3. 期待される動作:
   - Notion MCPが起動
   - データベースを検索
   - プロジェクト一覧を返す

4. 追加テスト:
「aidd-integration-systemプロジェクトの説明を教えて」

5. 書き込みテスト:
「新しいプロジェクトをNotionに追加して。
名前: テストプロジェクト
説明: MCP動作確認用」
```

#### 完了条件

- [ ] Integration作成完了
- [ ] データベース作成・共有完了
- [ ] MCP設定完了
- [ ] Cursor再起動完了
- [ ] 読み込みテスト成功
- [ ] 書き込みテスト成功

## Day 3-4: @Docs試験導入（⭐⭐⭐⭐ スコア4.1）

### 技術的根拠

**なぜ2番目か**:
1. **動画で推奨**: Docs活用で最新仕様を参照
2. **低コスト**: 15分の登録のみ
3. **効果測定可能**: 1週間で判断可能
4. **低リスク**: 効果なければ削除するだけ

---

### 実行手順

#### Step 1: 対象ドキュメント選定（3件）

**基準**: よく使うライブラリ、頻繁に参照する規約

**推奨候補**:

```markdown
1. **Python PEP 8**（Pythonコーディング規約）
   - URL: https://peps.python.org/pep-0008/
   - 理由: Python開発の標準、参照頻度高

2. **よく使うライブラリのドキュメント**
   - 例: Requests, FastAPI, Pandas等
   - 現在のプロジェクトで使用中のもの

3. **開発手法**
   - テスト駆動開発（TDD）
   - RESTful API設計ガイド
```

**ユーザーの選択**:
- 自分がよく使うライブラリを3つ選ぶ
- バージョン番号を確認

#### Step 2: Cursorへ登録（10分）

```markdown
各ドキュメントについて:

1. Cursor → `@` → `Docs` → `New Doc`

2. 設定内容:
   - Name: "Python PEP 8 (2024)"  ← バージョン番号含める
   - URL: https://peps.python.org/pep-0008/
   - Prefix: （同じURLでOK）

3. 「Add Doc」をクリック

4. 2-3件も同様に登録
```

#### Step 3: 動作確認（5分）

```markdown
1. テストプロンプト:
「@docs Python PEP 8 を参照して、
この関数をPEP 8に従ってリファクタリングしてください」

2. 期待される動作:
   - PEP 8の内容を参照
   - 規約に従ったコード生成

3. 参照漏れテスト（@なし）:
「この関数をリファクタリングしてください」

→ @ありとなしで品質差があるか確認
```

#### 完了条件

- [ ] 3件のドキュメント登録完了
- [ ] @ありでの動作確認成功
- [ ] @なしとの品質差を確認
