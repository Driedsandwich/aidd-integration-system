# AI駆動開発を10倍快適にする実践ガイド

## 概要

本ガイドは、YouTube動画「【実践】AI駆動開発を10倍快適にする【AIDD】」（数理の弾丸⚡️京大博士のAI解説、2025年6月27日公開）と、実際のワークスペースでの実装経験を基に作成された実践的なAI駆動開発手法の記録です。

## 背景

AI駆動開発において以下の4つの主要な問題が頻繁に発生します：

1. **AIエージェントが同じミスを繰り返す**
2. **書いたコードがカオスになってわけがわからなくなる**
3. **ライブラリの最新仕様を理解していない**
4. **欲しい機能がAIエージェントにない**

## 4つの解決アプローチ

### 1. ルール改善のサイクルを作る

#### 問題の詳細
- 仮想環境を使わずにグローバルインストールしてしまう
- 同じエラーを繰り返し発生させる
- ユーザーの意図を正しく理解できない

#### 解決策の核心
**完璧なルールを一度で作るのではなく、使用しながら改善する仕組みを構築する**

#### 実装方法

##### ルールの種類
1. **ユーザールール（プロジェクト共通）**
   - フルネーム、パーソナリティ等の基本設定
   - プロジェクトを問わず適用される

2. **プロジェクトルール（プロジェクト固有）**
   - プロジェクトごとの開発方針
   - `cursor rules generate`コマンドで編集可能

##### 推奨するルール構造
```
# Cursor Rules (project-level)

## General（共通）
- このファイルは「AIが同じ失敗を繰り返さない」ための学習ノート
- 不具合や学びが出たら、下の各章に短く追記する

## Environment（環境）
- 仮想環境を使う（例：Pythonなら `venv` / `uv`）
- 危険操作（削除・上書き）は必ず確認質問をする

## Implementation（実装）
- 変更時はテストまたは動作確認手順も提示する
- エラー時は原因候補・再現手順・修正案をセットで示す

## GitHub（運用）
- 作業はIssue単位で行う
- 1PR=1トピック。変更理由と範囲を明記する

## Requirements（要件）
- このプロジェクトで「やらないこと」も明記する

## Knowledge（知見の棚）
- 失敗例：グローバルに依存パッケージを入れて壊した
- 再発防止ルール：仮想環境を必須とする
```

##### ルール改善サイクル
1. **指示理解** - ユーザーからの指示を受け取る
2. **コード実行** - ファイル編集、Webサーチ等の作業実行
3. **エラー発生** - 問題が発生した場合のフィードバック
4. **解決策整理** - 問題と解決策を整理
5. **ルール反映** - 学習内容をルールに追加
6. **次の指示** - 改善されたルールで次の作業へ

#### 重要なポイント
- **ローカルファイルとしてルールを管理**できることが最大の強み
- フィードバックは人間から与えるか、AIが自己内省する
- 一定のやり取りが終わった後に必ずルールに反映する

### 2. Issueによる開発管理

#### 問題の詳細
- チャット内で作業を完結させてしまう
- コードがカオスになり、何をしているかわからなくなる
- 後から振り返りができない

#### 解決策の核心
**チャット内での作業完結を避け、GitHubのIssueを活用したPDCAサイクル**

#### 実装方法

##### PDCAサイクルの各段階

1. **Plan（計画）**
   - 機能要件の定義
   - テストケースの設計
   - Issueで計画を明文化

2. **Do（実行）**
   - 実装の進行
   - 進捗のIssueへの報告

3. **Check（評価）**
   - テスト実行と動作確認
   - 想定通りに実装できているかの検証

4. **Act（改善）**
   - 問題があれば計画の見直し
   - ルールの改善

##### GitHub MCPの活用
- Issue作成、ステータス管理、コメント追加の自動化
- マークダウンファイルと異なり、ステータス管理機能が内蔵
- 電車乗りながらでもレビュー可能

##### 具体的なワークフロー例
```
1. 新機能のIssue作成
   - タイトル：「WebUI機能実装」
   - 内容：実装する機能の詳細
   - テストケースの定義

2. 計画のレビューと承認
   - 認識合わせ
   - 必要に応じて計画の修正

3. 実装フェーズ
   - 進捗の定期的な報告
   - 問題発生時の相談

4. テストと評価
   - テストコードの実行
   - 動作確認

5. 完了処理
   - 最終コメントの追加
   - Issueのクローズ
```

#### 重要なポイント
- **ステータス管理機能**が内蔵されている点が重要
- スレッド形式で時系列に沿ったメッセージ管理が可能
- コード全体を汚すことなく、開発プロセスを管理

### 3. ドキュメントの活用

#### 問題の詳細
- AIエージェントがライブラリの最新仕様を理解していない
- 古い書き方や存在しないクラスを使用する
- 環境セットアップを正しく行わない

#### 解決策の核心
**Cursorの@マーク機能でドキュメントを参照し、最新情報を活用**

#### 実装方法

##### @マーク機能の活用
1. Cursorの@マークをクリック
2. 使用ライブラリのドキュメントを選択
3. 必要なドキュメントを参照リストに追加

##### 登録可能なドキュメント
- **ライブラリドキュメント**: LangChain、React、PyTest等
- **ツールドキュメント**: UV（仮想環境管理）、Plotly等
- **開発手法**: TDD（テスト駆動開発）等の記事
- **コーディング規約**: PEP8等
- **ウェブサイト**: 任意のURLを登録可能

##### 設定方法
1. 名前設定
2. リンク設定
3. プレフィックス設定（エントリーポイント）

#### 重要なポイント
- ライブラリドキュメントだけでなく**ウェブサイトのリンクも登録可能**
- 開発スタイルやコーディング規約も登録可能
- AIエージェントが事前にドキュメントを参照してから作業開始

### 4. MCP（Model Context Protocol）の導入

#### 問題の詳細
- AIエージェントに欲しい機能がない
- Webサーチ機能が欲しい
- GitHubアクセス機能が欲しい
- カスタム機能を追加したい

#### 解決策の核心
**MCPを使用して外部システムと連携し、AIエージェントの機能を拡張**

#### 実装方法

##### MCPの基本概念
- **MCPホスト**: AIが動いているアプリケーション（Cursor等）
- **MCPクライアント**: ホスト内に実装されている機能
- **MCPサーバー**: 外部システムとの連携を提供

##### MCPサーバーの自作
- **FastAPI風のインターフェース**で簡単に作成可能
- **所要時間**: 約40分程度で実装可能
- **例**: WebサーチMCP（Gemini API活用）

##### 実装例
```python
# WebサーチMCPの例
@app.post("/search")
async def search(query: str):
    # Gemini APIを使用して検索実行
    result = gemini_api.search(query)
    return {"result": result}
```

##### 設定方法
1. MCPサーバーの実装
2. 設定ファイルへの追加
3. カーソルでの機能利用

#### 重要なポイント
- **自分で拡張機能をパッと作れる**点が最大の利点
- GitHub連携、Webサーチ等の機能を簡単に追加可能
- エージェントの強みがバンバン生きる

## 実装のベストプラクティス

### 1. ルール改善の継続
- 作業終了時に必ず学習内容をルールに反映
- 失敗例と再発防止ルールを明確に記録
- 定期的なルールの見直しと更新

### 2. Issue管理の徹底
- すべての作業をIssueで管理
- 計画段階での認識合わせを重視
- ステータス管理を適切に行う

### 3. ドキュメントの充実
- 使用するライブラリのドキュメントを事前登録
- 開発手法や規約も含めて包括的に管理
- 定期的なドキュメントの更新

### 4. MCPの段階的導入
- 必要に応じてMCPサーバーを自作
- 小さく始めて徐々に機能を拡張
- 自作機能の効果測定と改善

## 効果測定の指標

### 定量的指標
- **開発速度の向上**: 機能実装時間の短縮
- **エラー発生率の減少**: 同じエラーの繰り返し発生の減少
- **コード品質の改善**: レビュー時間の短縮、バグ数の減少

### 定性的指標
- **開発体験の向上**: ストレスの減少、満足度の向上
- **学習効果**: 新しい技術の習得速度
- **チーム協働**: 知識共有の促進

## 注意点と制限事項

### 動画で言及された制限
- スピーカーの吉田氏は研究目的でのコーディング経験が中心
- プロダクト開発の経験は限定的
- 1ヶ月程度の実践経験に基づく手法

### 検証が必要な点
- 大規模チーム開発での適用性
- エンタープライズ環境での実用性
- 長期的な効果の検証

## まとめ

AI駆動開発を効率化する4つのアプローチを組み合わせることで、以下の効果が期待できます：

1. **ルール改善サイクル**: 同じミスの繰り返しを防止
2. **Issue管理**: 開発プロセスの透明性と効率性の向上
3. **ドキュメント活用**: 最新情報に基づく正確な実装
4. **MCP導入**: 必要な機能の拡張とカスタマイズ

これらの手法は、特に**ローカルファイルとしてルールやドキュメントを管理できる**ローカルAIエディタの強みを最大限活用するものです。

継続的な改善と最新情報の取り入れにより、AI駆動開発の効果を最大化することができます。
