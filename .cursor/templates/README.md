# Cursor テンプレート集

このディレクトリには、AI駆動開発で使用する各種テンプレートが含まれています。

---

## 📄 利用可能なテンプレート

### **1. ACE-based Rule Update Template**
**ファイル**: `ace-rule-update-template.md`  
**用途**: ルール更新時のACE 3段階プロセス適用  
**使用タイミング**: ルールファイル更新時

**プロセス**:
```
Stage 1: Reflect（振り返り）
  ↓
Stage 2: Generate Delta（差分生成）
  ↓
Stage 3: Apply（適用 - search_replace使用）
```

**効果**: コンテキスト崩壊の回避、詳細情報の保持

---

### **2. Concise Report Template**
**ファイル**: `concise-report-template.md`  
**用途**: 簡潔版完了報告書（50行以内）  
**使用タイミング**: タスク完了時の報告書作成

**構成**:
- 概要（目的・成果・所要時間）
- 主要成果（実施内容・ファイル・Issues）
- 定量データ（検証済み）
- 重要な学習（1-2点）
- 次のステップ
- 参照リンク

**効果**: 
- 報告書作成の効率化
- 必要最小限の情報記録
- 技術的合理性とハルシネーション回避を担保

**詳細版との使い分け**:
- **簡潔版**: 通常のタスク完了報告（50行以内）
- **詳細版**: 初回実施、重要なプロセス確立（200-300行）

---

## 📝 使用方法

### **テンプレートのコピー**
```bash
# 簡潔版報告書
cp .cursor/templates/concise-report-template.md docs/YYYY-MM-DD-project-name-report.md

# ACEルール更新
cp .cursor/templates/ace-rule-update-template.md [作業用ファイル名].md
```

### **テンプレートの編集**
1. `[プレースホルダー]`を実際の内容に置き換え
2. 不要なセクションは削除
3. 必要に応じてセクション追加

## 🎯 ベストプラクティス

### **簡潔版報告書**
- ✅ 50行以内を目標
- ✅ 定量データは必ず検証
- ✅ 学習ポイントは1-2点に絞る
- ✅ 技術的合理性を明記

### **ACEルール更新**
- ✅ Reflectで詳細に分析
- ✅ Generate Deltaで具体的な変更箇所を特定
- ✅ Applyでは`search_replace`を使用
- ✅ `write`による全体書き換えを避ける
