# Taskmaster Rules（無効化）

## 無効化理由

**2025-10-12**: 複雑性の観点から導入を見送り

### 判断根拠

**過剰な要素**:
- 40以上のコマンド体系
- タグシステム（複数コンテキスト管理）
- tasks.jsonの複雑な管理
- 非エンジニアには学習コスト高

**代替手段**:
- GitHub Issue管理で同等機能を実現
- PRD → Issue → 実装のシンプルなフロー
- 既存の`general.mdc`、`git.mdc`で十分

### 抽出した有用な設計思想

以下の思想は他のルールに反映済み：

1. **PRD駆動開発**
   - 要件定義 → 実装のフロー
   - `implementation.mdc`に反映

2. **MVP vs Production**
   - プロトタイプ: 速度優先
   - 本番: 品質優先
   - `implementation.mdc`に反映

3. **段階的な複雑性管理**
   - シンプル → 必要に応じて高度化
   - `general.mdc`に反映

### 将来の再評価条件

以下の状況が発生した場合は再検討：

1. **大規模化**
   - 10以上の並行プロジェクト
   - 複雑な依存関係の可視化が必要

2. **チーム拡大**
   - 統一されたタスク管理フォーマットが必要
   - 標準化されたPRD駆動開発

3. **既存システムの限界**
   - GitHub Issueでは管理しきれない
   - より高度なタスク管理が必要

### 保持理由

完全削除ではなく`.disabled/`に保持：
- 将来の再評価用の参照資料
- 有用な設計思想の保存
- Git履歴の明確化
