# サブPC Cursor導入プロンプト集

## 🎯 概要

サブアカウントのサブPCのCursorで、マルチデバイス・マルチアカウントシステムを導入するための適切なプロンプト集です。

## 📝 基本プロンプト

### **完全導入プロンプト**
```
以下のGitHubリポジトリをクローンし、マルチデバイス・マルチアカウントでのAI駆動開発統合システムを導入してください。

リポジトリURL: https://github.com/0nyx-lab/aidd-integration-system

## 導入要件

### 1. リポジトリのクローンとセットアップ
- リポジトリをクローン
- Python仮想環境の作成・有効化
- 依存関係のインストール

### 2. Cursor設定の同期
- .cursor/rules/ ディレクトリの作成
- templates/cursor-rules/ からルールファイルをコピー
- デバイス固有の設定（Mac環境）の適用

### 3. マルチデバイス同期システムの設定
- multi_device_config.json の設定
- デバイス情報の登録（sub-pc, macOS, sub-account）
- 同期システムのテスト実行

### 4. 知識蓄積システムの統合
- knowledge.mdc でのデバイス固有知見の記録
- GitHub同期の設定
- 自動同期スクリプトの作成

## 期待される結果
- メインPC（Windows）との知識共有が可能
- デバイス固有の知見を蓄積・共有
- 統合PDCAサイクルの実行
- 継続的な知識蓄積・学習システムの運用

詳細な実装手順は、リポジトリ内の「マルチデバイス実装ガイド.md」を参照してください。
```

## 🔍 トラブルシューティング用プロンプト

### **一般的な問題解決**
```
マルチデバイス同期システムで問題が発生した場合、以下の確認を行ってください：

1. GitHub権限の確認
   - サブアカウントがコラボレーターとして追加されているか
   - Write権限が付与されているか

2. 環境設定の確認
   - 仮想環境が正しく有効化されているか
   - 依存関係が正しくインストールされているか

3. 同期システムの確認
   - multi_device_sync.py が正常に実行されるか
   - 設定ファイルが正しく作成されているか

4. ネットワーク接続の確認
   - GitHubへの接続が可能か
   - 認証情報が正しく設定されているか

詳細なトラブルシューティングは、実装ガイドの「📞 トラブルシューティング」を参照してください。
```

### **権限エラーの解決**
```
GitHub権限エラーが発生した場合：

1. リポジトリのアクセス確認
   ```bash
   git remote -v
   ```

2. 認証の再設定
   ```bash
   git config --global user.name "Your Name"
   git config --global user.email "your.email@example.com"
   ```

3. SSH鍵の設定確認
   ```bash
   ssh -T git@github.com
   ```

4. メインアカウントでのコラボレーター追加確認
```

### **同期エラーの解決**
```
同期エラーが発生した場合：

1. ネットワーク接続の確認
   ```bash
   ping github.com
   ```

2. Git設定の確認
   ```bash
   git config --list
   ```

3. 手動同期の実行
   ```bash
   git pull origin main
   git push origin main
   ```

4. 同期システムの再実行
   ```bash
   python3 multi_device_sync.py
   ```

### **仮想環境エラーの解決**
```
仮想環境エラーが発生した場合：

1. 仮想環境の再作成
   ```bash
   rm -rf .venv
   python3 -m venv .venv
   source .venv/bin/activate
   pip install -r requirements.txt
   ```

2. Pythonバージョンの確認
   ```bash
   python3 --version
   ```

3. pipのアップグレード
   ```bash
   pip install --upgrade pip
   ```

4. 依存関係の再インストール
   ```bash
   pip install -r requirements.txt
   ```
```
